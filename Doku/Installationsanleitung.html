🚀 IFAS-Assistent Installation & Setup

Schnellstart-Anleitung für neue Systeme

Diese Anleitung führt Sie Schritt für Schritt durch die Einrichtung des IFAS-Assistenten auf einem neuen Windows-Rechner. Das System stützt sich auf lokale KI (LMStudio) und Texterkennung (Tesseract OCR).

🔧 1. System-Voraussetzungen (Python & Tesseract)

1a

Python installieren

Der IFAS-Assistent benötigt Python 3.10 oder neuer.

  1. Laden Sie Python herunter: ⬇️ Python für Windows laden
  2. Führen Sie den Installer aus.
  3. WICHTIG: Setzen Sie den Haken bei "Add Python 3.x to PATH" unten im Installationsfenster, bevor Sie auf "Install Now" klicken!
Testen: Öffnen Sie ein Terminal (CMD oder PowerShell) und tippen Sie python --version. Es sollte eine Version wie Python 3.11.x erscheinen.
1b

Tesseract OCR installieren

Für die Analyse von eingescannten PDFs oder Bildern benötigen Sie Tesseract OCR.

  1. Laden Sie den Windows-Installer herunter (z.B. UB-Mannheim Build): ⬇️ Tesseract Installer
  2. Installieren Sie Tesseract. Der Standardpfad ist meist C:\Program Files\Tesseract-OCR.
  3. Öffnen Sie in Windows die "Systemumgebungsvariablen bearbeiten".
  4. Klicken Sie auf "Umgebungsvariablen".
  5. Suchen Sie unter "Systemvariablen" die Variable Path, klicken Sie auf Bearbeiten und fügen Sie den Tesseract-Ordnerpfad (z.B. C:\Program Files\Tesseract-OCR) als neuen Eintrag hinzu.

🧠 2. Lokale KI-Engine (LMStudio) einrichten

Der IFAS-Assistent nutzt ein lokales Large Language Model, das über LMStudio gehostet wird. Das schützt den Datenschutz, da keine Dokumentengeninhalte an Cloud-Anbieter gesendet werden.

2

LMStudio installieren & Modell laden

  1. Laden Sie LMStudio herunter: ⬇️ LMStudio laden
  2. Installieren und starten Sie LMStudio.
  3. Suchen Sie in der Suchleiste von LMStudio nach einem geeigneten Modell.
    Empfohlen: Ein leichtgewichtiges Instruction-Modell wie Llama-3-8B-Instruct-GGUF oder Mistral-7B-Instruct (quantisiert, z.B. Q4_K_M für optimale Geschwindigkeit).
  4. Klicken Sie bei dem gewünschten Modell auf Download.

Lokalen API-Server starten

  1. Wechseln Sie in LMStudio zum Tab "Local Server" (Schnittstellen-Icon links in der Leiste).
  2. Wählen Sie oben im Dropdown-Menü das gerade heruntergeladene Modell aus.
  3. Achten Sie darauf, dass der Port auf 1234 steht (Standard).
  4. Klicken Sie auf Start Server.
Hinweis: Der LMStudio Server muss laufen, damit der IFAS-Assistent Dokumente analysieren kann! Alternativ kann das Backend von LMStudio auch per CLI oder Auto-Start-Skript gestartet werden.

🚀 3. IFAS-Assistent Starten

Sobald Python, Tesseract und LMStudio bereit sind, können Sie die eigentliche Anwendung starten.

3

Anwendung konfigurieren & starten

  1. Öffnen Sie eine PowerShell oder Eingabeaufforderung im Ordner IFAS-Assistent.
  2. (Optional, aber empfohlen) Erstellen Sie eine virtuelle Umgebung: 
    python -m venv venv
.\venv\Scripts\activate
  3. Installieren Sie die Abhängigkeiten aus der requirements.txt: 
    pip install -r requirements.txt
  4. Initialisieren Sie die Datenbank (nur beim allerersten Start nötig): 
    flask db upgrade
python seed.py   # (Falls ein Seed-Skript vorhanden ist, um Standard-Kategorien anzulegen)
  5. Starten Sie die Anwendung: 
    python run.py
🎉 Fertig! Die Web-Oberfläche ist nun unter http://localhost:5050 erreichbar. Unten rechts im Windows-Systemtray finden Sie zudem das IFAS-Icon, über das Sie das Dashboard ebenfalls direkt aufrufen können.

⚙️ 4. Erste Schritte

Wenn Sie das Dashboard öffnen, loggen Sie sich als Administrator ein (Standardzugangsdaten erfragen oder neu anlegen lassen).

Prüfen Sie im Dashboard unter Systemstatus, ob alle Verbindungen grün sind:

  • LMStudio API: Verbunden? (Wenn rot: Prüfen Sie in LMStudio, ob der Server läuft.)
  • IFAS API: Verbunden? (Wenn rot: Prüfen Sie den API-Key in der Konfiguration oder ob der Mock-Modus läuft.)

Navigieren Sie danach zu den Einstellungen, um die Überwachungsordner sowie die E-Mail-Postfächer zu konfigurieren.
