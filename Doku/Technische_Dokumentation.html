📋 Systemübersicht

Der IFAS-Assistent ist eine webbasierte Anwendung zur automatisierten Verarbeitung von Posteingängen für das IFAS-System (Informationssystem für Arbeitsschutz). Das System überwacht verschiedene Eingangsquellen (E-Mail-Postfächer, lokale Ordner, Netzwerkfreigaben), extrahiert automatisch Dokumente und deren Inhalte, analysiert diese mittels eines lokalen KI-Modells (LMStudio) und ermöglicht die validierte Übertragung in das IFAS-Fachverfahren.

Kernfunktionen

📨 Multi-Source Posteingang
Automatische Überwachung und Import von Dokumenten aus IMAP-Postfächern, lokalen Ordnern und Netzlaufwerken. Duplikat-Erkennung über Content-Hashing.
🤖 KI-gestützte Analyse
Dokumentklassifikation und Datenextraktion mittels LMStudio (lokales LLM). Keyword-basierter Fallback bei LLM-Ausfall. Konfidenz-Scoring pro Feld.
✅ Validierung & Korrektur
Side-by-Side-Ansicht: Originaldokument neben extrahierten Daten. Manuelle Korrektur einzelner Felder, Kategorie-Änderung und Betriebsstätten-Zuordnung.
📤 IFAS-Übertragung
Direkte Übertragung der validierten Daten in das IFAS-Fachverfahren über die IFAS-API. Erstellung von Anzeigen (Vorgängen) mit Dokumenten-Anhängen.
👥 Benutzerverwaltung
Rollenbasierte Zugriffskontrolle (Admin / Sachbearbeiter). Login-Tracking, Audit-Log, E-Mail-Benachrichtigungen bei Systemereignissen.
⚙️ Konfigurierbare Kategorien
Frei definierbare Dokumentkategorien mit eigenen Feldern, Keywords und IFAS-Mappings. Felder sind typisiert (Text, Datum, Auswahl, etc.).

🗄️ Datenmodell

Die Datenbank verwendet SQLite über SQLAlchemy ORM. Alle 8 Modelle sind in app/models.py definiert.

User

Benutzermodell mit rollenbasierter Zugriffskontrolle.

Feld Typ Beschreibung
id Integer (PK) Primärschlüssel
username String(64) Eindeutiger Benutzername
email String(120) E-Mail-Adresse
password_hash String(256) Gehashtes Passwort (Werkzeug)
role String(20) admin oder user
display_name String(100) Anzeigename
is_active Boolean Konto aktiv/deaktiviert
notification_preferences Text (JSON) E-Mail-Benachrichtigungseinstellungen

Category

Dokumentkategorien definieren die Klassifikation (z.B. Mutterschutz, Unfallanzeige, Pyrotechnische Anzeige).

Feld Typ Beschreibung
name String(100) Interner Name der Kategorie
display_name String(150) Anzeigename
icon / color String Emoji-Icon und Hex-Farbe
ifas_art String(100) IFAS Vorgangsart für API-Mapping
ifas_zusatzart String(100) IFAS Zusatzart (optional)
keywords Text Komma-getrennte Keywords für KI-Matching
fields Relationship → CategoryField (1:n, cascade delete)

CategoryField

Dynamische Formularfelder pro Kategorie. Jedes Feld hat Typ, Validierung und KI-Steuerung.

Feld Typ Beschreibung
label String(100) Anzeigebeschriftung
key String(50) Interner Schlüssel für Daten-Mapping
content_id_ifas String(100) IFAS Content-ID für Sonderaktion-API
field_type String(20) text, number, date, select, textarea
options Text (JSON) Auswahloptionen (bei Typ select)
auto_fill Boolean KI darf dieses Feld automatisch befüllen
show_in_validation Boolean In der Validierungsansicht sichtbar
position Integer Sortierposition im Formular

InputSource

Konfiguration der Eingangsquellen mit typenspezifischen JSON-Einstellungen.

Feld Typ Beschreibung
source_type String(20) imap, folder, network, ifas_api
config_json Text Typenspezifische Konfiguration als JSON
is_active Boolean Polling aktiv/inaktiv
auto_analyze Boolean Automatische KI-Analyse nach Import
poll_interval Integer Polling-Intervall in Sekunden
trash_folder String(100) Zielordner für gelöschte E-Mails
processed_folder String(100) Zielordner nach IFAS-Übertragung

Document

Zentrale Entität — jedes importierte Dokument durchläuft den Status-Lifecycle:

pending → analyzing → ready → transferred oder rejected

Feld Typ Beschreibung
external_id String(100) UUID für Dateizuordnung
filename String(255) Originaler Dateiname
stored_path String(500) Pfad zur gespeicherten Datei
status String(20) Verarbeitungsstatus (s. Lifecycle)
raw_text Text Extrahierter Volltext
category_id FK → Category Von KI erkannte Kategorie
ai_confidence Float KI-Konfidenz (0.0 – 1.0)
extracted_data_json Text (JSON) Extrahierte Felddaten
content_hash String(64) SHA256-Hash zur Duplikat-Erkennung
imap_uid Integer IMAP-UID für E-Mail-Operationen
ifas_aktenzeichen String(100) IFAS-Aktenzeichen nach Übertragung
analysis_started_at DateTime Beginn KI-Analyse (Performance-Tracking)

Attachment

E-Mail-Anhänge oder eingebettete Dateien eines Dokuments.

AuditLog

Protokollierung aller relevanten Benutzeraktionen (Login, Übertragung, Ablehnung, Konfigurationsänderungen).

AppConfig

Key-Value-Store für Anwendungsweite Einstellungen (LLM-URL, OCR-Pfad, SMTP-Konfiguration etc.).

⚡ Service-Schicht

Die gesamte Geschäftslogik ist in 7 eigenständige Service-Module aufgeteilt:

📧 EmailConnector email_connector.py
Aufgabe: IMAP-Postfach-Anbindung und E-Mail-Import.
Methoden:
  • poll() — Neue E-Mails per IMAP-UID abrufen
  • _import_email(msg, uid) — E-Mail in System importieren
  • move_to_folder(doc, folder) — E-Mail verschieben (IMAP COPY + DELETE)
  • test_connection() — Verbindungstest mit Statistiken
  • get_folders() — Verfügbare IMAP-Ordner auflisten
Duplikat-Erkennung: SHA256-Hash des E-Mail-Inhalts wird mit bestehenden verglichen.
📁 FolderMonitor folder_monitor.py
Aufgabe: Überwachung lokaler und Netzwerk-Ordner.
Methoden:
  • poll() — Ordner auf neue Dateien scannen
  • _import_document(path, name) — Datei importieren & kopieren
  • move_to_processed/rejected/trash(doc) — Datei verschieben
Unterstützte Formate: PDF, DOCX, DOC, ODT, MSG, EML
Features: Datei-Hash-Prüfung, MSG/EML-Parsing mit Attachment-Extraktion.
📄 DocumentExtractor document_extractor.py
Aufgabe: Unified Text-Extraktion aus verschiedenen Formaten.
Formate: PDF (PyMuPDF + OCR), DOCX (python-docx), ODT (odfpy), MSG (extract-msg), EML (email-Modul).
Features: Automatische Tesseract-Erkennung unter Windows, Fallback-Strategien.
🔬 PDFExtractor pdf_extractor.py
Aufgabe: Spezialisierte PDF-Verarbeitung.
Strategien:
  • standard — Textlayer-Extraktion inkl. Formularfelder (Widgets)
  • tesseract — OCR mit 2x-Zoom für bessere Qualität
Extras: Tabellenextraktion via pdfplumber, seitenweise Extraktion.
🤖 LMStudioService lmstudio_service.py
Aufgabe: KI-gestützte Dokumentenanalyse via LMStudio.
Methoden:
  • analyze_document(text, attachments, categories) — Hauptanalyse
  • _build_analysis_prompt() — Dynamischer Prompt aus Kategorien/Feldern
  • _parse_response() — JSON-Parsing mit aggressivem Fallback
  • check_connection() — LMStudio-Erreichbarkeit prüfen
  • reload_model() — Modell bei Absturz neu laden
Fallback: Keyword-basierte Analyse wenn LLM nicht erreichbar.
⚠️ API-Endpunkte (LMStudio):
GET /api/v1/models — Modelle auflisten
POST /api/v1/chat — Chat-Completions
POST /api/v1/models/load — Modell laden

Analyse-Prompt

Der Prompt wird dynamisch aus Kategorien und Feldern zusammengebaut. Erkennt das LLM die Daten nicht, wird eine Keyword-basierte Hybrid-Konfidenz zur Fallback-Validierung genutzt.

🔗 IfasApiClient ifas_api_client.py
Aufgabe: IFAS-Fachverfahren Integration.
Methoden:
  • search_betriebsstaette(query) — Betriebsstätten suchen
  • create_betriebsstaette(data) — Neue Betriebsstätte anlegen
  • create_anzeige(anzeige_data) — Vorgang/Anzeige anlegen
  • attach_document(aktenzeichen, path) — Dokument anhängen
  • create_post(post_data) — Posteingang erstellen
  • get_arbeitskorb(status) — Items aus dem IFAS Arbeitskorb lesen
  • update_post_status(post_id, status) — Status eines Posteingangs ändern
Modi: Mock-Modus für Entwicklung, API-Modus für Produktion.
⚠️ Hinweis: Die IFAS-API wird aktuell von der Firma Kisters zu einer REST-API umgebaut. Die hier dokumentierten Endpunkte basieren auf dem aktuellen Stand und werden sich erweitern.
🚀 Zukünftige Endpunkte (Geplant):
In kommenden Versionen werden weitere Endpunkte ergänzt zur:
- Abfrage von Posteingängen direkt aus dem IFAS Arbeitskorb
- Weiterleitung eines Posteingangs (IFAS-seitig)
- Verwaltung der Mitarbeiterzuständigkeit

1. search_betriebsstaette(query, ort, plz, limit)

Sucht nach Betriebsstätten im IFAS-System.

Parameter Typ Beschreibung
query str Suchbegriff (Name des Betriebs)
ort str (opt) Ort-Filter
limit int Max. Ergebnisse 
# API-Endpunkt (Produktion):
# GET {IFAS_API_URL}/betriebsstaetten?q=Musterfirma&limit=10
# Header: Authorization: Bearer {API_KEY}

2. create_betriebsstaette(data)

Legt eine neue Betriebsstätte in IFAS an.

Parameter Typ Beschreibung
data dict Stammdaten (name, ort, plz, strasse, sachbearbeiter) 
# POST {IFAS_API_URL}/betriebe
# Erwartet: {"success": true, "bs_nr": "BS-..."}

3. create_anzeige(anzeige_data)

Erstellt einen neuen Vorgang (Anzeige) im IFAS-System.

Feld Typ Beschreibung
art str IFAS Vorgangsart
bs_nr str Betriebsstättennummer
fields dict content_id → Wert 
# Payload-Struktur:
payload = {
    "art": "Mutterschutz",
    "status": "Entwurf",
    "bs_nr": "BS-12345",
    "sonderaktion_values": {
        "CONTENT_ID_001": "Max Mustermann"
    }
}
# POST {IFAS_API_URL}/anzeigen
💡 Hinweis: sonderaktion_values werden über CategoryField.content_id_ifas gemappt.

4. attach_document(aktenzeichen, file_path)

# POST {IFAS_API_URL}/anzeigen/{aktenzeichen}/dokumente
# Body: multipart/form-data mit Datei

5. create_post(post_data)

Erstellt einen reinen Posteingang im System (ApiPost).

# POST {IFAS_API_URL}/post
# Body: JSON (z.B. betreff, eingangsdatum)

6. get_arbeitskorb(status='NEU')

Liest ausstehende oder bearbeitete Elemente aus dem Arbeitskorb.

# GET {IFAS_API_URL}/arbeitskorb?status=NEU

7. update_post_status(post_id, status='DMS')

Aktualisiert den Status eines verarbeiteten Posteingangs.

# PUT {IFAS_API_URL}/post/{post_id}/status
📬 EmailService email_service.py
Aufgabe: SMTP-basierte Alert-Benachrichtigungen.
Funktion: send_alert_to_subscribers(subject, body, alert_type)
Alert-Typen: error (Systemfehler), pending_overflow (zu viele wartende Dokumente).
Empfänger werden anhand der Benutzer-Präferenzen bestimmt. Fallback auf Admin-E-Mail.

Scheduler (scheduler.py)

Der APScheduler steuert zwei Hintergrundaufgaben:

Job Intervall Funktion
Source Polling Konfig. pro Quelle (Standard: 60s) Alle aktiven InputSources abfragen, neue Dokumente importieren. Bei auto_analyze=True wird sofort die KI-Analyse gestartet.
Background Analysis On-Demand Alle Dokumente mit Status pending analysieren. Thread-Safe über analysis_lock. Watchdog für "stuck" Dokumente (>5 Min in analyzing).
